# Telegram Bot для управления проектами Grad Service

Бот для управления проектами, задачами, документами и встречами с проверкой адресов через ГИС.

## 📋 Возможности

### Для клиентов:
- Создание заявки на проект
- Загрузка исходных файлов
- Просмотр статуса проекта
- Получение готовых документов
- Обратная связь по проекту

### Для руководителей:
- Просмотр и регистрация заявок
- Назначение исполнителей на проекты
- Создание задач для исполнителей
- Формирование календарного плана с этапами
- Управление встречами (создание, подтверждение, завершение)
- Просмотр статистики и загруженности
- Управление клиентской базой

### Для исполнителей:
- Просмотр назначенных задач
- Загрузка документов в проект
- Отметка выполнения задач
- Обратная связь

### Общее:
- ✅ Уведомления о новых задачах, дедлайнах, встречах
- 📅 Календарь совещаний
- 👤 Профиль пользователя
- 🔍 Поиск проектов и документов
- 🗺️ **Проверка адресов встреч через ГИС** (Яндекс.Геокодер)

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка

Создайте файл `.env` в корне проекта:

```bash
copy .env.example .env
```

Отредактируйте `.env` и добавьте ваши данные:

```env
# Токен бота (получить у @BotFather)
BOT_TOKEN=your_bot_token_here

# ID администраторов (через запятую)
ADMIN_IDS=123456789

# API ключ Яндекс.Геокодера (для проверки адресов встреч)
YANDEX_GEOCODER_API_KEY=your_yandex_api_key_here
```

**Получение токена бота:**
1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot`
3. Следуйте инструкциям
4. Скопируйте полученный токен в `.env`

**Получение API ключа Яндекс.Геокодера:**
1. Зайдите в [Кабинет разработчика Яндекс](https://developer.tech.yandex.ru/)
2. Нажмите "Подключить API" → "JavaScript API и HTTP-геокодер"
3. Создайте ключ и скопируйте его в `.env`

### 3. Запуск бота

```bash
python main.py
```

## 🗄️ Модели данных

| Модель | Описание |
|--------|----------|
| **User** | Пользователи (клиент, руководитель, исполнитель) |
| **Project** | Проекты с статусами и дедлайнами |
| **Task** | Задачи по проектам с исполнителями |
| **ProjectStage** | Этапы календарного плана |
| **Document** | Документы проектов и задач |
| **Meeting** | Совещания (онлайн/офлайн) |
| **MeetingParticipant** | Участники совещаний со статусами |
| **Feedback** | Обратная связь по проектам |
| **Notification** | Уведомления пользователей |
| **ClientCompany** | Клиентские компании |
| **GISCheckLog** | Логи проверок адресов через ГИС |

Модели реализованы с использованием **SQLAlchemy 2.0** (Mapped attributes).

## 🗺️ ГИС (Географическая информационная система)

Бот проверяет адреса встреч на принадлежность к зоне обслуживания:

1. **Геокодинг**: Адрес → Координаты (Яндекс.Геокодер API)
2. **Проверка зоны**: Координаты внутри GeoJSON полигона?
3. **Результат**: 
   - ✅ Внутри зоны — встреча одобряется
   - ⚠️ Снаружи зоны — рекомендуется онлайн-формат

Файл зоны: `data/service_zone.geojson`
